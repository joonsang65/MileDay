# M6. 위젯형 데스크톱 캘린더와 인증 UI 고도화

## 문서 목적

M6 단계에서는 M5에서 구현한 Electron 캘린더 UI를 데스크톱 위젯처럼 사용할 수 있도록 개선한다.
앱은 일반 데스크톱 애플리케이션처럼 큰 창과 타이틀바를 갖는 형태가 아니라, 사용자의 홈 화면에 캘린더만 떠 있는 위젯 형태로 동작해야 한다.

또한 첫 실행 화면은 로그인만 제공하지 않고, 신규 사용자가 바로 회원가입을 진행할 수 있는 진입점을 함께 제공해야 한다.

## 회원가입 UX 기준

첫 실행 인증 화면은 로그인과 회원가입을 같은 화면 안에서 전환하는 구조로 구현한다.
별도 라우팅을 만들기보다 현재 `AuthPanel` 컴포넌트 안에서 모드를 전환하는 방식이 M5 구조와 가장 잘 맞는다.

권장 동작은 다음과 같다.

1. 기본 모드는 로그인이다.
2. 사용자가 `회원가입` 버튼을 누르면 회원가입 폼으로 전환한다.
3. 회원가입 폼에는 이메일, 비밀번호, 비밀번호 확인 입력을 둔다.
4. 비밀번호는 백엔드 스키마 기준과 동일하게 최소 8자를 요구한다.
5. 비밀번호와 비밀번호 확인이 다르면 API 호출 전에 오류를 표시한다.
6. 회원가입 성공 후에는 로그인 화면으로 전환하고 성공 메시지를 표시한다.
7. 자동 로그인은 M6 범위에서는 필수가 아니다. Supabase 이메일 확인 정책에 따라 즉시 로그인 가능 여부가 달라질 수 있기 때문이다.

## API 연동 기준

프론트엔드 API 클라이언트에 다음 메서드를 추가한다.

```ts
signup(email: string, password: string): Promise<SignUpResult>
```

요청 API:

```http
POST /auth/signup
Content-Type: application/json
```

요청 본문:

```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

성공 응답:

```json
{
  "success": true,
  "data": {
    "user_id": "uuid",
    "email": "user@example.com"
  }
}
```

## 위젯 창 구현 기준

M6의 핵심은 Electron 창을 일반 앱 창이 아니라 데스크톱 위젯처럼 만드는 것이다.
이를 위해 `frontend/electron/main.ts`의 `BrowserWindow` 설정을 조정한다.

권장 설정 방향:

```ts
new BrowserWindow({
  width: 900,
  height: 620,
  frame: false,
  resizable: false,
  minimizable: false,
  maximizable: false,
  fullscreenable: false,
  skipTaskbar: true,
  show: false,
  backgroundColor: "#F7F8FA",
});
```

세부 기준:

- `frame: false`로 OS 기본 타이틀바와 닫기, 최소화, 최대화 버튼을 제거한다.
- 창 내부에도 별도 닫기, 최소화 버튼을 기본 노출하지 않는다.
- `alwaysOnTop: true`는 기본 사용하지 않는다. 다른 앱 위에 계속 떠 있으면 요구사항 3을 위반하기 때문이다.
- 위젯이 바탕화면에서는 보이되 Chrome 같은 일반 창 위에는 올라오지 않아야 한다.
- 사용자가 창을 이동할 수 있어야 하는 경우 캘린더 상단 일부 영역에 `-webkit-app-region: drag`를 적용할 수 있다.
- 버튼, 입력, 날짜 셀처럼 클릭 가능한 영역에는 `-webkit-app-region: no-drag`를 적용한다.

## 데스크톱 상주 동작 해석

요구사항 3은 다음처럼 해석한다.

> 홈 화면에는 항상 떠 있지만, 사용자가 Chrome 같은 다른 앱 창을 열거나 활성화하면 그 창 위로 MileDay가 덮이지 않는다.

즉, MileDay는 “항상 위” 앱이 아니다.
일반 창의 z-order를 따르되, 창 자체를 자동 종료하거나 숨기지 않는다.
사용자가 바탕화면을 다시 보면 MileDay 창이 남아 있어야 한다.

구현 우선순위:

1. `alwaysOnTop`을 사용하지 않는 프레임리스 위젯 창으로 구현한다.
2. `skipTaskbar` 적용 여부를 확인한다.
3. Windows 환경에서 바탕화면 위젯처럼 보이는지 수동 검증한다.
4. 필요할 경우 후속 단계에서 트레이 아이콘 또는 설정 메뉴를 추가한다.

주의:

- Electron만으로 Windows 바탕화면의 아이콘 레이어에 완전히 붙는 “진짜 데스크톱 위젯”을 안정적으로 구현하는 것은 제한이 있다.
- M6에서는 OS 내부 WorkerW/Progman 핸들 조작 같은 네이티브 확장은 사용하지 않는다.
- 요구사항을 만족하는 1차 구현은 “프레임 없는 일반 창이 바탕화면에 남아 있고, 다른 앱보다 위에 강제로 올라오지 않는 상태”로 정의한다.

## UI 레이아웃 변경 기준

M5 UI는 일반 앱 화면에 가까운 구조였으므로 M6에서는 더 작은 위젯 밀도로 조정한다.

변경 기준:

- 전체 화면을 채우는 `app-shell` 느낌을 줄이고, 창 전체를 캘린더 위젯 표면으로 사용한다.
- 사이드 패널이 캘린더를 지나치게 좁히면 기본 화면에서는 월간 캘린더 중심으로 표시한다.
- Today List와 날짜 상세는 접이식 패널, 하단 패널, 또는 선택 날짜 영역으로 축약한다.
- 월간 캘린더는 첨부 이미지처럼 날짜 셀 안에 일정 텍스트가 바로 보이는 형태를 우선한다.
- 헤더는 날짜, 이전/다음, 오늘, 월/주 전환 정도만 남긴다.
- 로그아웃은 항상 보이는 큰 버튼보다 작은 메뉴 또는 아이콘 버튼으로 제공한다.

## 종료와 숨김 정책

기본 창 버튼을 제거하면 사용자가 앱을 종료할 경로가 필요하다.
M6에서 다음 중 하나를 선택해 구현한다.

권장안:

- 창 내부 우측 상단에 작고 낮은 강조도의 메뉴 버튼을 둔다.
- 메뉴 안에 `로그아웃`, `앱 숨기기`, `앱 종료`를 제공한다.
- 앱 종료는 Electron main process에 IPC를 연결해 처리한다.

대안:

- 시스템 트레이 아이콘을 추가하고 트레이 메뉴에서 `열기`, `숨기기`, `종료`를 제공한다.

M6 MVP에서는 트레이보다 내부 메뉴가 구현 범위와 테스트 부담이 작다.
다만 `skipTaskbar: true`를 적용하면 숨긴 창을 다시 여는 경로가 필요하므로, 숨김 기능을 넣는 경우 트레이 구현을 함께 고려해야 한다.

| --- | --- |

| 인증 UI | 로그인 모드와 회원가입 모드가 전환되는지 검증 |
| 인증 UI | 회원가입 비밀번호 확인 불일치 시 API를 호출하지 않는지 검증 |
| 인증 UI | 회원가입 성공 후 로그인 화면으로 돌아가고 성공 메시지를 표시하는지 검증 |
| 인증 UI | 로그인 실패와 회원가입 실패 메시지를 각각 표시하는지 검증 |
| Electron main | `BrowserWindow`가 `frame: false`, `resizable: false`, `skipTaskbar` 등 위젯 설정으로 생성되는지 검증 가능한 구조로 분리 |

## 구현 계획

1. `frontend/src/api/types.ts`에 회원가입 응답 타입을 추가한다.
2. `frontend/src/api/client.ts`에 `signup` 메서드를 추가한다.
3. `AuthPanel`을 로그인/회원가입 모드 전환 구조로 수정한다.
4. `App`에 `handleSignup` 흐름과 성공 메시지 상태를 추가한다.
5. `frontend/electron/main.ts`의 `BrowserWindow` 설정을 위젯 창 기준으로 변경한다.
6. 위젯 크기에 맞게 `frontend/src/styles.css`의 레이아웃 밀도와 최소 크기를 조정한다.
7. 필요하면 Electron 창 옵션 생성을 함수로 분리해 테스트 가능하게 만든다.
8. 프론트엔드 테스트에 회원가입 플로우와 위젯 UI 회귀 케이스를 추가한다.
9. `npm test`, `npm run build`, `npm run lint`를 실행한다.
10. Windows에서 실제 Electron 개발 실행으로 위젯 창 동작을 수동 검증한다.

## 완료 기준

- 첫 실행 인증 화면에서 로그인과 회원가입을 모두 수행할 수 있다.
- 회원가입 입력값 검증이 프론트엔드에서 동작한다.
- 회원가입 성공과 실패 상태가 명확히 표시된다.
- Electron 창에 OS 기본 닫기, 최소화, 최대화 영역이 보이지 않는다.
- 앱 화면이 일반 데스크톱 앱보다 위젯형 캘린더에 가깝게 축소되어 보인다.
- Chrome 같은 다른 창을 활성화했을 때 MileDay가 그 위를 덮지 않는다.
- 바탕화면을 다시 볼 때 MileDay 창이 유지된다.
- 프론트엔드 테스트, 빌드, 린트가 통과한다.
- 구현 중 추가한 주석은 한글로 작성한다.
