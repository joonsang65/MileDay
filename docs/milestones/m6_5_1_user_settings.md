# M6.5.1. 사용자 설정 API와 설정 패널

## 목표

M6.5.1 단계에서는 계정 기준 사용자 설정을 실제 DB에 저장하고, 저장된 설정이 캘린더 UI의 기본 동작에 바로 반영되도록 구현한다.

헤더의 로그아웃 버튼 위치에는 설정 버튼을 두고, 로그아웃은 설정 패널 하단으로 이동한다.

## 구현 범위

| 영역 | 내용 |
|---|---|
| Settings API | 현재 사용자 기준 설정 조회와 부분 수정 |
| 설정 기본값 | 설정 row가 없으면 기본값을 생성한 뒤 반환 |
| 캘린더 기본 view | `calendar_view` 값으로 월간/주간 기본 화면 결정 |
| 주 시작 요일 | `week_starts_on` 값으로 캘린더 그리드와 요일 라벨 변경 |
| 휴일 표현 | 한국 공휴일 API 결과와 `holiday_display` 값으로 날짜 칸 표시 변경 |
| 언어 | `language` 값으로 핵심 UI 문구를 `ko`/`en` 전환 |
| 로그아웃 위치 | 설정 패널 하단에 로그아웃 버튼 배치 |

## API 계약

### GET `/settings`

현재 사용자의 설정을 조회한다.

설정 row가 없으면 서버가 기본값 row를 생성하고 그 값을 반환한다.

### PATCH `/settings`

전달된 필드만 수정한다.

클라이언트가 전달한 `user_id`는 사용하지 않고, JWT에서 추출한 현재 사용자 ID만 사용한다.

### 허용값

| 필드 | 허용값 |
|---|---|
| `calendar_view` | `month`, `week` |
| `holiday_display` | `normal`, `weekend_like`, `hidden` |
| `week_starts_on` | `0`, `1` |
| `language` | `ko`, `en` |

## 공휴일 처리

공휴일 제공자는 공공데이터포털의 `한국천문연구원_특일 정보` API를 기준으로 한다.

백엔드 환경 변수:

| 변수 | 설명 |
|---|---|
| `HOLIDAY_API_SERVICE_KEY` | 공공데이터포털 API 서비스 키 |
| `HOLIDAY_API_BASE_URL` | 공휴일 API 호출 URL. 기본값은 `https://apis.data.go.kr/B090041/openapi/service/SpcdeInfoService/getHoliDeInfo` |

API 키가 없거나 외부 API 호출이 실패하면 캘린더 요청은 실패하지 않는다.

fallback 기준:

- 공휴일 정보는 비워 둔다.
- 토요일/일요일은 프론트에서 주말로 계속 표시한다.
- 설정 패널 또는 트러블슈팅 문서에서 API 키 누락과 외부 API 실패 가능성을 확인할 수 있게 한다.

`holiday_display` 동작:

| 값 | 동작 |
|---|---|
| `normal` | 공휴일 이름과 휴일 스타일을 표시 |
| `weekend_like` | 공휴일 이름은 숨기고 주말과 같은 스타일로 표시 |
| `hidden` | 공휴일 이름과 별도 휴일 스타일을 숨김 |

## UI 흐름

1. 사용자가 헤더의 설정 버튼을 누른다.
2. 오른쪽 사이드 영역이 설정 패널로 전환된다.
3. 사용자는 기본 캘린더 view, 휴일 표현 방식, 주 시작 요일, 언어를 수정한다.
4. 저장 성공 시 설정은 즉시 앱 상태에 반영된다.
5. 설정 패널 하단의 로그아웃 버튼으로 로그아웃할 수 있다.
6. 설정 패널을 닫으면 기존 Today List, 날짜 상세, 빠른 추가 패널이 다시 보인다.

## 검증 기준

- 인증 없는 `/settings` 요청은 `401`을 반환한다.
- 설정 조회 시 row가 없으면 기본 설정이 생성되고 반환된다.
- 설정 수정 시 허용되지 않는 enum 값은 `400`으로 거부된다.
- `week_starts_on=1`이면 월요일부터, `0`이면 일요일부터 캘린더가 시작한다.
- `calendar_view=week` 저장 후 기본 view가 주간 화면으로 바뀐다.
- `language=ko/en` 변경 후 핵심 UI 문구가 즉시 바뀐다.
- 공휴일 API 키가 없어도 캘린더 조회는 성공한다.
- 공휴일 API 성공 시 날짜 칸에 공휴일 표시가 반영된다.
