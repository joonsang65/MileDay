# MileDay - 요구사항 명세 및 마일스톤 DB 정리

## 문서 메타데이터

- Notion 데이터베이스: 요구사항 명세 및 마일스톤
- 상위 페이지: 마일스톤 계획 및 개발 환경 정의
- 기준 시점: 2026-07-05
- 정리 목적: Codex가 구현 범위와 우선순위를 Notion 요구사항 기준으로 해석하도록 하기 위함

## 1. 데이터베이스 스키마

| 속성 | 타입 | 설명 |

| 마일스톤 | title | 마일스톤 이름 또는 상세 요구사항 ID |
| 구분 | select | 마일스톤 / 상세 요구사항 / 설명 |

## 2. ID 설명

| ID Prefix | 의미 | 적용 범위 |
|---|---|---|
| BE | Backend | FastAPI 서버, API, CORS, 로그, 응답 형식 |
| DB | Database | Supabase PostgreSQL 테이블, 사용자별 데이터 저장, 삭제 정책 |
| SC | Security | RLS, 민감 정보 보호, 인증 토큰, 권한 검증 |
| MEM | Member | 회원가입, 로그인, 자동 로그인, 로그아웃, 사용자별 접근 |
| GL | Goal | 목표 생성, 조회, 수정, 삭제, 상세 조회, 반복 목표 |
| RP | Repeat | 반복 일정 생성, 반복 예외, 공휴일 처리 |
| MS | Milestone | 마일스톤 생성, 조회, 수정, 삭제, 완료 처리 |
| CAL | Calendar | 월간/주간 캘린더, 날짜 이동, 날짜 강조, 일정 표시 |
| TODO | Today List | 오늘 예정 마일스톤, 부모 목표 표시, 완료 처리, 빈 상태 |
| VIEW | Date Detail | 날짜별 목표/마일스톤 조회와 날짜 상세 내 완료 처리 |
| WD | Widget Desktop | Electron 위젯 창, 위치/크기/투명도/자동 실행 |
| FE | Frontend | UI/UX, 색상, 글꼴, 로딩 상태 |
| ERR | Error | 입력값 검증, 오류 메시지, 빈 데이터, 공통 예외 처리 |
| AUTO | Automation | 자연어 일정 수정, AI 마일스톤, 자동 배치, 리스케줄링 |
| INT | Integration | 외부 캘린더 연동, 동기화 방향, 이벤트 매핑 |
| REL | Release | Electron 패키징, 모바일 앱, 실행 환경 문서화 |

## 4. 상세 요구사항

### M1. 개발 환경 구축하기

| ID | 분류 | 기능명 | 기능 상세 |
|---|---|---|---|
| BE-01 | 백엔드 | FastAPI 서버 구성 | 백엔드는 Python FastAPI 기반으로 구현되며, 프론트엔드 요청을 처리한다 |
| BE-02 | 백엔드 | API 기반 데이터 처리 | 프론트엔드는 Supabase에 직접 접근하지 않고 FastAPI API를 통해 데이터를 처리한다 |
| BE-06 | 백엔드 | CORS 설정 | 프론트엔드에서 FastAPI 백엔드로 API 요청을 보낼 수 있도록 CORS를 설정한다 |
| BE-07 | 백엔드 | API 응답 통일 | 백엔드는 성공 및 실패 응답을 일관된 형식으로 반환한다 |
| BE-08 | 백엔드 | 백엔드 로그 기록 | 백엔드는 API 요청 처리 중 발생한 오류와 주요 실행 정보를 로그로 기록한다 |
| BE-09 | 백엔드 | 에러 로그 분리 | 백엔드는 일반 실행 로그와 에러 로그를 구분하여 확인할 수 있도록 관리한다 |
| DB-01 | 데이터베이스 | 사용자 데이터 저장 | 목표와 마일스톤 데이터는 Supabase Postgres에 저장된다 |
| DB-02 | 데이터베이스 | 목표 테이블 관리 | goals 테이블은 목표 제목, 마감일, 반복 여부, 반복 유형을 저장한다 |
| DB-03 | 데이터베이스 | 마일스톤 테이블 관리 | milestones 테이블은 마일스톤 제목, 수행 예정일, 완료 여부, 목표 ID를 저장한다 |
| DB-04 | 데이터베이스 | 사용자별 데이터 분리 | 각 목표와 마일스톤은 user_id를 기준으로 사용자별로 분리 저장된다 |
| DB-05 | 데이터베이스 | 목표 삭제 시 마일스톤 삭제 | 목표가 삭제되면 해당 목표에 속한 마일스톤도 함께 삭제된다 |
| ERR-04 | 오류 처리 | 예외 처리 공통화 | 백엔드는 공통 예외 처리 구조를 통해 API 오류를 일관된 응답과 로그로 관리한다 |
| SC-01 | 보안 | RLS 적용 | Supabase Row Level Security를 적용하여 사용자가 본인 데이터만 접근하도록 제한한다 |
| SC-02 | 보안 | 민감 정보 보호 | Supabase service role key, DB URL 등 민감 정보는 프론트엔드에 노출하지 않는다 |
| SC-04 | 보안 | 백엔드 권한 검증 | 백엔드는 사용자가 요청한 목표 또는 마일스톤이 본인 소유인지 검증한다 |

## 5. 구현 범위 해석 기준

- M1~M7은 MVP 구현에 필요한 핵심 범위로 본다
