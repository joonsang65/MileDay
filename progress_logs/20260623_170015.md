# Supabase 마이그레이션 시스템 작업 로그 - 2026-06-23 17:00:15

## 작업 범위

- Supabase 연동을 위한 환경변수 예시 구조를 정리했다.
- SKILL.md에 정의된 `goals`, `milestones` 테이블 생성 SQL을 추가했다.
- Row Level Security 활성화 SQL을 추가했다.
- 사용자별 접근 제한 정책 SQL을 추가했다.
- `updated_at` 자동 갱신 트리거 SQL을 추가했다.
- SQL 파일을 파일명 순서대로 실행하는 Python migration runner를 구현했다.

## 생성 및 수정한 파일

- `.env.example`
- `migrations/001_create_base_tables.sql`
- `migrations/002_enable_rls.sql`
- `migrations/003_create_policies.sql`
- `migrations/004_create_updated_at_trigger.sql`
- `scripts/migrate.py`
- `scripts/requirements.txt`

## 환경변수 구조

- Electron/Vite 앱에서 사용하는 공개 키:
  - `VITE_SUPABASE_URL`
  - `VITE_SUPABASE_ANON_KEY`
- 로컬 migration script에서만 사용하는 DB 접속 문자열:
  - `SUPABASE_DB_URL`

`SUPABASE_DB_URL`은 Electron renderer 또는 패키징된 앱에 노출하면 안 된다.

## Migration Runner 동작

- `.env`에서 `SUPABASE_DB_URL`을 읽는다.
- `public.schema_migrations` 테이블을 생성한다.
- `migrations/*.sql` 파일을 이름순으로 정렬한다.
- 이미 실행된 migration 파일은 건너뛴다.
- 새 migration을 실행한 뒤 `schema_migrations`에 파일명을 기록한다.
- 성공 시 commit하고, 실패 시 rollback한다.

## 검증 결과

- `python -m py_compile .\scripts\migrate.py`: 통과
- 실제 Supabase DB migration 실행은 `.env`에 실제 `SUPABASE_DB_URL`이 필요하므로 수행하지 않았다.
